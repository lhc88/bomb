## 🛠️ 기술 구조

### 아키텍처
```
BombermanGame (FlameGame)
├─ GridMap (생성, 파괴, 좌표 관리)
├─ PlayerComponent (이동, 폭탄, 피해)
├─ EnemyComponent[] (AI, 이동, 피해)
├─ BossComponent (패턴, 이동, 피해)
├─ BombComponent[] (타이머, 폭발)
└─ ExplosionComponent[] (범위, 효과, 정리)
```

### 의존성
```
GameScreen (UI)
    ↓
BombermanGame (Logic)
    ↓
GridMap (Map + Tiles)
    ↓
Components (Player, Enemy, Bomb, Explosion)
```

### 데이터 흐름
```
입력 → GameScreen
    → BombermanGame.handle*Input()
        → PlayerComponent.tryMove/tryPlaceBomb()
            → GridMap 상태 변경
                → ExplosionComponent 생성
                    → Enemy.takeDamage()
                        → 점수 계산
```
